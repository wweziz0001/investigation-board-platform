# [2.0.1] - 2025-03-06

## 🔧 الإصلاحات

### 1. إصلاح ترتيب وسائط دالة hasProjectAccess

**المشكلة:**
جميع API routes كانت تُرجع خطأ 403 Forbidden عند محاولة الوصول للمشاريع، رغم أن المستخدم عضو في المشروع.

**السبب:**
ترتيب الوسائط في استدعاء دالة `hasProjectAccess` كان خاطئاً في جميع ملفات API.

توقيع الدالة:
```typescript
hasProjectAccess(userId, projectId, requiredRole)
```

الاستخدام الخاطئ:
```typescript
hasProjectAccess(projectId, userId, requiredRole)  // ❌
```

الاستخدام الصحيح:
```typescript
hasProjectAccess(userId, projectId, requiredRole)  // ✅
```

**الملفات المتأثرة:**
- `src/app/api/projects/[id]/route.ts` - إصلاح 3 استدعاءات
- `src/app/api/events/route.ts` - إصلاح استدعاءين
- `src/app/api/events/[id]/route.ts` - إصلاح 3 استدعاءات
- `src/app/api/relationships/route.ts` - إصلاح استدعاءين
- `src/app/api/relationships/[id]/route.ts` - إصلاح 3 استدعاءات
- `src/app/api/evidence/[id]/route.ts` - إصلاح 3 استدعاءات
- `src/app/api/comments/[id]/route.ts` - إصلاح 3 استدعاءات
- `src/app/api/projects/[id]/members/route.ts` - إصلاح استدعاءين
- `src/app/api/projects/[id]/members/[memberId]/route.ts` - إصلاح استدعاءين

**النتيجة:**
- ✅ API يعيد الآن 200 OK بدلاً من 403 Forbidden
- ✅ المستخدمون يمكنهم الوصول لمشاريعهم
- ✅ لوحة التحقيق تفتح بشكل صحيح

### 2. إصلاح قراءة البيانات في project-store

**المشكلة:**
الـ store لا يقرأ البيانات بشكل صحيح من استجابة API.

**السبب:**
API يعيد البيانات في `data.data` لكن الـ store كان يبحث في `data.project` و `data.events`.

**الملفات المتأثرة:**
- `src/stores/project-store.ts` - تصحيح قراءة البيانات

**النتيجة:**
- ✅ تحميل المشروع بشكل صحيح
- ✅ تحميل الأحداث والعلاقات
