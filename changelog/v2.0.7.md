# v2.0.7

## Bug Fixes

### Fixed Project Dropdown Menu Actions
- **Issue**: Project dropdown menu items (Edit Project, Manage Members, Share, Archive, Delete) were not working
- **Cause**: DropdownMenuItem components had no onClick handlers attached
- **Solution**: Added onClick handlers and implemented dialogs for each action

### Changes Made

1. **Edit Project Dialog**
   - Added form with name, description, status, and priority fields
   - Implemented PUT request to update project details

2. **Manage Members Dialog**
   - Shows list of current project members with roles
   - Allows adding new members by email
   - Supports removing members (except owners)
   - Shows role icons (Crown for Owner, Shield for Admin, User for Member)

3. **Share Dialog**
   - Displays project share link
   - Copy to clipboard functionality

4. **Archive Action**
   - Sets project as archived and redirects to home

5. **Delete Action**
   - Shows confirmation dialog
   - Permanently deletes project and all associated data

### New Files
- `src/app/api/users/search/route.ts` - User search endpoint for adding members

### Updated Files
- `src/app/projects/[id]/page.tsx` - Added dialogs and handlers for all dropdown actions
