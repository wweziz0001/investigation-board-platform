# [2.0.5] - 2025-03-06

## 🎨 تحسينات التصميم

### تحسين صيغة عرض التاريخ

**المشكلة:**
التاريخ كان يظهر بصيغة غير واضحة تعتمد على إعدادات المتصفح المحلية (toLocaleDateString)، مما قد يسبب ارتباكاً للمستخدمين.

**الحل:**
توحيد صيغة عرض التاريخ إلى `dd/MM/yyyy` (مثال: 06/03/2026) في جميع أنحاء التطبيق باستخدام مكتبة `date-fns`.

**التغييرات:**

### 1. event-dialog.tsx
```typescript
// قبل الإصلاح
{formData.eventDate ? format(formData.eventDate, 'PPP') : 'Select date'}

// بعد الإصلاح
{formData.eventDate ? format(formData.eventDate, 'dd/MM/yyyy') : 'Select date'}
```

### 2. event-node.tsx
```typescript
// قبل الإصلاح
{new Date(nodeData.eventDate).toLocaleDateString()}

// بعد الإصلاح
{(() => {
  try {
    const date = typeof nodeData.eventDate === 'string'
      ? parseISO(nodeData.eventDate)
      : new Date(nodeData.eventDate);
    return format(date, 'dd/MM/yyyy');
  } catch {
    return nodeData.eventDate;
  }
})()}
```

**الملفات المتأثرة:**
- `src/components/board/event-dialog.tsx` - تحديث صيغة التاريخ في منتقي التاريخ
- `src/components/board/event-node.tsx` - تحديث صيغة التاريخ في عقدة الحدث

**النتيجة:**
- ✅ صيغة تاريخ موحدة وواضحة (dd/MM/yyyy)
- ✅ لا اعتماد على إعدادات المتصفح المحلية
- ✅ معالجة أخطاء تحويل التاريخ بشكل آمن
