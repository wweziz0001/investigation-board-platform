# v2.0.6

## Bug Fixes

### Fixed Event Node Action Buttons
- **Issue**: Edit, Connect, and Delete buttons in event nodes were not working
- **Cause**: Buttons had no onClick handlers attached
- **Solution**: Added callback props (onEdit, onConnect, onDelete) to EventNodeData interface and implemented onClick handlers for each action button
- **Files Changed**:
  - `src/components/board/event-node.tsx` - Added callback props and onClick handlers
  - `src/components/board/investigation-board.tsx` - Pass callback functions when creating nodes

### Changes Made
1. Extended `EventNodeData` interface to include:
   - `onEdit?: () => void` - Opens edit dialog for the event
   - `onConnect?: () => void` - Initiates connection mode
   - `onDelete?: () => void` - Deletes the event after confirmation

2. Updated hover actions in EventNode:
   - Edit button now opens the EventDialog with the event data
   - Connect button starts connection mode and prompts user to select target
   - Delete button shows confirmation dialog and removes the event

3. Updated InvestigationBoard to pass callbacks:
   - Each node receives proper handlers that interact with the store
